# 智能提醒助手 (Smart Reminder) - AI Agent 指南

本指南专为 AI 编程助手（如 Cursor、Aider、Copilot 等）提供项目上下文、开发规范与工作流约束。在修改、生成或重构代码前，请务必严格遵循本文件中的规范。

## 一、项目概述与技术栈

- **项目名称**：智能提醒助手 (Smart Reminder)
- **业务背景**：旨在提升教育类应用中学生学习任务的完成率与推送打开率，包含学生端的学习任务呈现与浮窗提醒（Study Buddy Widget），以及后端的智能策略匹配与管理后台看板。
- **技术栈**：
  - 核心框架：Next.js (App Router), React
  - 样式方案：Tailwind CSS v4
  - 交互动画：Framer Motion
  - UI 组件：shadcn/ui
  - 状态管理：Zustand (全局客户端状态), SWR (服务端获取)

## 二、基础环境与运行命令

- **包管理器**：pnpm
- **安装依赖**：`pnpm install`
- **启动开发环境**：`pnpm dev`
- **构建生产包**：`pnpm build`

## 三、强制编码规范 (CRITICAL RULES)

**AI Agent 必须无条件遵守以下核心原则和代码规范**，确保代码库的安全、统一与高性能：

### 3.1 基本原则与项目规范
- **最小改动**：只改与需求直接相关的代码，不要顺手进行不必要的全局重构。
- **保留注释**：绝不删除或修改与当次变更无关的代码注释和文档。
- **组件单一职责**：单个文件行数**不得超过 300 行**，超出请拆分组件或抽离逻辑。

### 3.2 React & Hooks 最佳实践
- **禁止使用 `useEffect` / `useLayoutEffect`**：对于副作用，优先使用事件回调、派生计算或 SWR 的生命周期回调替代。
- **禁止手写 `useMemo` / `useCallback`**：项目依赖 React Compiler 自动优化，不再需要手动包裹优化 Hook。

### 3.3 状态管理与数据请求
- **服务端数据**：强制使用 SWR 或 React Query 进行获取与缓存，**禁止**在组件内直接调用 `fetch` 或 `axios`（需通过 service 层封装的 Hook 调用）。
- **组件内简单状态**：使用 `useState`。
- **复杂或深层嵌套数据更新**：**强制**使用 `immer`（如 `produce` 或 `useImmer`），特别是针对 2 层以上的嵌套展开。
- **高频交互状态**：使用 Signals 或 Zustand。
- **URL 状态**：使用 `nuqs` / `useSearchParams`。
- **全局状态限制**：如非必要，禁止新建 `Context`。直接操作 `localStorage` 必须通过封装的 storage 工具访问。

### 3.4 类型与安全 (TypeScript)
- **显式类型**：**完全禁止**使用 `any` 类型。若类型暂时未知，请使用 `unknown` 并配合类型守卫（Type Guards）替代。
- **禁止使用强制断言**：避免使用 `as` 强制断言，应使用类型守卫或 Zod 等校验库。
- **禁止危险操作**：无例外地禁止使用 `eval` / `new Function`。使用 `dangerouslySetInnerHTML` 之前必须先经过 sanitize 处理。
- **异常处理**：禁止写空的 `catch {}` 吞掉错误，至少需要记录日志或上报给监控。

### 3.5 样式规范 (Tailwind CSS v4)
- **禁用 `px` 单位**：className 中绝对**禁止出现 `px` 任意值**（唯一的例外是固定的 `1px`，此时必须写作 `w-px` 等预设类）。
- **禁止硬编码颜色**：禁用 hex/rgb 形式的硬编码色值，统一使用设计令牌类名（Design Tokens）。后端返回的动态色值允许使用内联 `style`，静态部分仍用 Tailwind。
- **禁止百分比任意值**：不允许出现 `w-[50%]` 的写法，必须改用内置的分数写法如 `w-1/2`。
- **条件类名合并**：统一使用 `cn()` 工具函数处理动态样式的组合。

### 3.6 性能与用户体验
- **长列表优化**：超过 50 项的列表必须实现虚拟滚动。
- **图片优化**：强制使用 `next/image` 并明确指定尺寸。
- **按需加载**：非首屏模块统一使用 `dynamic()` 或 `React.lazy()` 延迟加载。
- **请求瀑布流**：避免请求瀑布，利用并行请求或独立的 SWR Hook 处理数据依赖。
- **加载态提示**：优先使用 Skeleton 骨架屏提供局部加载反馈，**禁止**使用全局级别的粗暴 "Loading..."。

## 四、代码修改自检清单（Self-Check）

AI 在每次生成或修改代码后，必须在内部自行核对以下清单（无需输出核对过程），确保没有违反底线：
- [ ] 无 `useEffect` / `useLayoutEffect`。
- [ ] 无手写 `useMemo` / `useCallback`。
- [ ] 无 `any` 类型声明或强制断言。
- [ ] Tailwind 类名中无 `px` 任意值及硬编码颜色。
- [ ] 深层更新均已使用 `immer`。
- [ ] 无吞掉错误的空 `catch`。
- [ ] 只修改了与需求相关的部分，保留了无关注释。
